## Notes
#### git submodules
If you use git submodules that are not required for building the project with `nix`, then you should add these directories to the `gitignoreSourcePure` list in `nix/komet-node/default.nix`, see the respective left-over `TODO` in the nix file. Otherwise, the nix build that is uploaded to the nix binary cache by CI might not match the version that is requested by `kup`, in case this project is offered as a package by `kup`. This happens because `git` and `nix` treat git submodules inconsistently: a submodule directory may exist and be empty, or not exist at all, and that difference changes the resulting nix hash. The hash matters when CI offers or downloads cached nix derivations. See [runtimeverification/k/pull/4804](https://github.com/runtimeverification/k/pull/4804) for more information.
